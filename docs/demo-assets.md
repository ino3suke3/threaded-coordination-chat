# Demo Assets Guidance

## Purpose

This document defines the minimum public-facing demo assets recommended for this repository.

## Recommended assets

- One landing screenshot showing the thread list, message timeline, and task panel
- One task-detail screenshot showing plan items and approvals
- One short GIF or MP4 showing:
  - selecting a message
  - creating a task
  - adding a plan item
  - resolving an approval

## Capture guidance

- Use neutral sample thread titles and message text
- Do not include customer names, product names, or internal project names
- Keep the capture focused on the generic coordination flow
- Prefer a browser width that shows all three columns at once

## Suggested filenames

- `docs/assets/thread-overview.png`
- `docs/assets/task-detail.png`
- `docs/assets/demo-flow.gif`

## Suggested README placement

- Put one static screenshot near the top of the README
- Link to the GIF or MP4 under a short "Demo" section
- Keep captions descriptive and generic
