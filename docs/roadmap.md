# Roadmap

## Purpose

This document groups the current non-MVP items into a clear public-facing roadmap.

## Phase 1: Local prototype hardening

- Improve component-level test coverage
- Add more seed data scenarios
- Refine empty states and validation feedback
- Add keyboard and accessibility polish

## Phase 2: Collaboration basics

- Introduce multi-user support
- Add identity and session handling
- Add shared persistence beyond browser localStorage
- Add conflict-safe state synchronization

## Phase 3: Integration hooks

- Define backend storage model
- Add repository synchronization strategy
- Add external event and automation adapters
- Add environment-aware configuration

## Phase 4: Operations readiness

- Add deployment workflow
- Add monitoring and logging hooks
- Add backup and restore guidance
- Add runbooks for support and recovery

## Not in roadmap by default

- Domain-specific workflows
- Product-locked terminology
- Private business logic
